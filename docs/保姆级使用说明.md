## 系统简介

志愿者排表系统是一个专门用于大型活动（如马拉松）志愿者管理的综合系统，主要功能包括：

- **志愿者信息管理**：支持普通志愿者、内部志愿者、家属志愿者、团体志愿者、情侣志愿者等多种类型
- **面试管理**：汇总面试打分、分离已面试和未面试人员
- **智能排班**：根据志愿者属性、岗位需求、特殊关系等进行智能分组和排班
- **数据验证**：自动检查数据完整性、重复信息、资格条件等
- **报表生成**：生成各种格式的排班表、统计报告、资格审查报告等

### 系统特点

✅ **交互式界面**：用户友好的菜单式操作界面
✅ **自动文件检查**：执行前自动检查所需文件是否存在
✅ **灵活配置**：通过配置文件管理所有路径和参数
✅ **详细日志**：完整的操作日志记录，便于问题排查
✅ **批量处理**：支持大量志愿者数据的批量处理

### 必需的输入文件详解

#### 1. 基础志愿者表格

**普通志愿者招募表.xlsx**
- **作用**：存储所有报名的普通志愿者信息
- **关键字段**：学号、姓名、性别、联系方式、意向岗位等
- **格式要求**：Excel格式（.xlsx），第一行为表头

**内部志愿者表.xlsx**
- **作用**：存储内部工作人员志愿者信息
- **特点**：通常可直接担任组长，无需面试
- **关键字段**：学号、姓名、部门、是否愿意担任组长等

**家属志愿者表.xlsx**
- **作用**：存储内部工作人员家属信息
- **特殊字段**：您是谁的家属（用于关联内部人员）
- **限制条件**：每个内部人员最多携带2名家属

**团体志愿者表.xlsx**
- **作用**：存储外部组织团体志愿者信息
- **特点**：通常按团体分配，需保持团体完整性

#### 2. 特殊关系表格

**情侣志愿者表.xlsx**
- **作用**：记录志愿者情侣关系
- **用途**：确保情侣分配到同一小组
- **字段**：情侣一学号、情侣一姓名、情侣二学号、情侣二姓名

#### 3. 岗位和管理表格

**岗位表.xlsx**
- **作用**：定义所有可用的工作岗位
- **字段**：岗位名称、岗位描述、需求人数、工作要求等

**直接委派名单.xlsx**
- **作用**：指定特定人员到特定岗位
- **用途**：优先级最高的人员分配

#### 4. 面试相关文件

**面试打分表/目录**
- **内容**：多个面试官的打分文件
- **格式**：每个文件包含多个志愿者的面试成绩
- **字段**：学号、姓名、各项得分、总分等

## 10个核心程序详解

### 面试结果收集模块

#### 程序1：汇总面试打分表
**功能描述**：将多个面试官的打分表合并为一个统一的成绩表

**输入文件**：
- 面试打分表目录（包含所有面试官的打分文件）

**输出文件**：
- `pipeline/01_interview_results/统一面试打分表.xlsx`

**执行条件**：
- 面试打分表目录存在
- 目录中至少有一个有效的Excel文件

**用途**：为后续的志愿者筛选提供统一的面试成绩数据

---

#### 程序2：分离已面试和未面试人员
**功能描述**：根据面试成绩表，从普通志愿者中分离出已面试和未面试的人员

**输入文件**：
- 普通志愿者招募表.xlsx
- 统一面试打分表.xlsx

**输出文件**：
- `pipeline/01_interview_results/普通志愿者表.xlsx`（已面试人员）
- `pipeline/01_interview_results/未面试人员名单.xlsx`（未面试人员）

**执行条件**：
- 程序1已成功执行
- 输入文件格式正确，包含学号、姓名等关键字段

**用途**：为排班阶段提供准确的已面试志愿者名单

---

### 排表模块

#### 程序3：基本信息核查和收集
**功能描述**：检查所有志愿者表格的数据完整性，发现重复记录，收集系统元数据

**输入文件**：
- 普通志愿者招募表.xlsx
- 内部志愿者表.xlsx
- 家属志愿者表.xlsx
- 团体志愿者目录（可选）
- 情侣志愿者表.xlsx（可选）
- 岗位表.xlsx
- 直接委派名单.xlsx（可选）

**输出文件**：
- `pipeline/02_scheduling_preparation/metadata.json`（系统元数据）
- `reports/多表格查重报告.txt`（重复记录检查报告）
- `reports/情侣志愿者资格核查结果报告.txt`（情侣资格报告）

**执行条件**：
- 所有输入文件存在
- 文件包含必要的字段（学号、姓名等）

**重要功能**：
- 检查跨表格的学号和姓名重复
- 生成志愿者总数统计
- 创建系统配置元数据

---

#### 程序4：正式普通志愿者和储备志愿者拆分
**功能描述**：根据面试成绩，将普通志愿者分为正式志愿者和储备志愿者

**输入文件**：
- 普通志愿者表.xlsx（程序2输出）
- metadata.json（程序3输出）
- 统一面试打分表.xlsx（程序1输出）

**输出文件**：
- `pipeline/02_scheduling_preparation/正式普通志愿者表.xlsx`
- `pipeline/02_scheduling_preparation/储备志愿者表.xlsx`

**执行条件**：
- 程序1、2、3已成功执行
- metadata.json中定义了正式志愿者人数

**分配规则**：
- 按照面试成绩从高到低排序
- 成绩相同者按报名顺序排列
- 根据metadata.json中的M值确定正式志愿者数量

---

#### 程序5：家属志愿者资格审查
**功能描述**：检查家属志愿者是否符合资格限制

**输入文件**：
- 家属志愿者表.xlsx

**输出文件**：
- `reports/家属志愿者资格审查结果报告.txt`

**执行条件**：
- 家属志愿者表存在且格式正确

**审查规则**：
- 每个内部志愿者最多携带2名家属
- 检查"您是谁的家属"字段的重复情况
- 生成违规名单和统计信息

---

#### 程序6：情侣志愿者资格核查
**功能描述**：检查情侣志愿者双方是否都符合参加资格

**输入文件**：
- 情侣志愿者表.xlsx
- 正式普通志愿者表.xlsx（程序4输出）
- 内部志愿者表.xlsx
- 家属志愿者表.xlsx
- 团体志愿者目录

**输出文件**：
- `reports/情侣志愿者资格核查结果报告.txt`

**执行条件**：
- 程序4已成功执行
- 情侣志愿者表包含完整的双方信息

**核查逻辑**：
- 检查情侣双方是否都在合格的志愿者列表中
- 生成不符合资格的情侣名单
- 为后续排班提供有效的情侣关系数据

---

#### 程序7：小组划分及组长分配
**功能描述**：根据岗位需求和志愿者属性，进行小组划分和组长分配

**输入文件**：
- 岗位表.xlsx
- 内部志愿者表.xlsx
- 正式普通志愿者表.xlsx（程序4输出）
- metadata.json（程序3输出）

**输出文件**：
- `pipeline/02_scheduling_preparation/小组信息表.xlsx`

**执行条件**：
- 程序3、4已成功执行
- 岗位表定义了小组数量和容量

**分配算法**：
- 优先从内部志愿者中选拔组长
- 确保每个小组都有指定的组长
- 考虑志愿者的专业背景和意向

---

#### 程序8：绑定集合生成
**功能描述**：创建各种绑定关系集合，确保特殊关系在排班中得到满足

**输入文件**：
- 情侣志愿者表.xlsx
- 家属志愿者表.xlsx
- 团体志愿者目录
- 直接委派名单.xlsx

**输出文件**：
- `pipeline/02_scheduling_preparation/绑定集合表.xlsx`
- `reports/绑定集合汇总报告.txt`

**执行条件**：
- 程序5、6已成功执行
- 所有关系数据已通过资格审查

**绑定类型**：
- 情侣绑定：确保情侣在同一小组
- 家属绑定：确保家属与关联人员在同一小组
- 团体绑定：保持团体成员的完整性
- 直接委派：满足指定的岗位分配要求

---

#### 程序9：排表主程序
**功能描述**：核心排班算法，将所有志愿者分配到具体的小组和岗位

**输入文件**：
- metadata.json（程序3输出）
- 小组信息表.xlsx（程序7输出）
- 绑定集合表.xlsx（程序8输出）

**输出文件**：
- `output/总表.xlsx`（包含所有志愿者的最终排班）

**执行条件**：
- 程序7、8已成功执行
- 所有必需的配置文件存在

**分配策略**：
1. 优先级排序：直接委派 > 绑定集合 > 组长 > 特殊角色 > 普通志愿者
2. 容量约束：确保每个小组不超过最大人数
3. 角色平衡：合理分配小闪电、摄影等特殊角色
4. 冲突解决：自动处理分配冲突和约束违反

---

#### 程序10：总表拆分和表格整合
**功能描述**：将总表拆分为各个小组的独立文件，并生成整合的大总表

**输入文件**：
- 总表.xlsx（程序9输出）
- metadata.json（程序3输出）
- 小组信息表.xlsx（程序7输出）
- 储备志愿者表.xlsx（程序4输出）

**输出文件**：
- `output/各小组名单/`目录下的小组文件（按小组号命名）
- `output/大总表.xlsx`（包含多个工作表的整合文件）

**执行条件**：
- 程序9已成功执行
- 总表包含完整的小组分配信息

**生成内容**：
- 每个小组的独立Excel文件
- 包含多个工作表的大总表：
  - 总表（带颜色标识）
  - 小组信息表
  - 储备志愿者表
  - 颜色对照表

### Q1: 系统启动时提示"模块不存在"怎么办？

### Q2: 输入文件格式有什么要求？

**A:** 基本要求如下：

1. **文件格式**：必须是Excel格式（.xlsx或.xls）
2. **编码**：建议使用UTF-8编码保存文本数据
3. **表头**：第一行必须是字段名称
4. **必需字段**：
   - 所有志愿者表：学号、姓名
   - 面试打分表：学号、姓名、各项得分
   - 岗位表：岗位名称、需求人数

### Q3: 程序执行失败怎么办？

**A:** 按以下步骤排查：

1. **查看错误信息**：仔细阅读控制台输出的错误信息
2. **检查日志文件**：查看 `logs/` 目录下的详细日志
3. **验证输入文件**：确认文件格式正确且未被损坏
4. **检查依赖关系**：确认前置程序已成功执行
5. **检查磁盘空间**：确保有足够的存储空间

### Q4: 如何处理大量数据？

**A:** 处理大数据的建议：

1. **分批处理**：如果志愿者超过1000人，考虑分批处理
2. **增加内存**：确保系统有足够的RAM（建议8GB+）
3. **优化存储**：使用SSD硬盘提高I/O性能
4. **清理临时文件**：定期清理 `pipeline/` 目录下的临时文件
5. **监控系统资源**：使用任务管理器监控CPU和内存使用

## 错误排查指南

### 常见错误类型及解决方案

#### 1. 文件不存在错误

**错误信息**：`FileNotFoundError: 文件不存在`

**可能原因**：
- 文件路径错误
- 文件名拼写错误
- 文件被移动或删除

**解决方案**：
```bash
# 检查文件是否存在
ls input/普通志愿者招募表.xlsx

# 使用系统检查功能
选择：4  # 系统前提条件检查
```

#### 2. 字段匹配错误

**错误信息**：`未找到必要的字段列`

**可能原因**：
- Excel表头字段名不符合预期
- 字段名包含特殊字符或空格

**解决方案**：
1. 检查表头字段名，确保包含：学号、姓名等关键字段
2. 修改 `config/config.yaml` 中的字段映射：
```yaml
field_mappings:
  student_id: "学号"  # 修改为实际的表头字段名
  name: "姓名"
```

#### 3. 内存不足错误

**错误信息**：`MemoryError` 或系统变慢

**解决方案**：
```bash
# 检查系统内存
python -c "import psutil; print(f'可用内存: {psutil.virtual_memory().available/1024/1024/1024:.1f}GB')"

# 清理临时文件
python simple_interactive.py
选择：7  # 清理临时文件选项
```

#### 4. 编码错误

**错误信息**：`UnicodeDecodeError`

**解决方案**：
1. 确保Excel文件使用UTF-8编码
2. 避免在字段名中使用特殊字符
3. 使用纯文本而不是富文本格式

#### 5. 权限错误

**错误信息**：`PermissionError`

**解决方案**：
1. 以管理员身份运行程序
2. 关闭Excel程序，确保文件未被锁定
3. 检查文件夹的读写权限
