# 志愿者排表系统 - 操作指南

## 第一部分：环境配置

### 1. Python 环境

**系统要求:**
- Python 3.8+
- Windows/Mac/Linux

**安装步骤:**

1. 打开`cmd`或`PowerShell`，进入项目文件夹
```bash
cd 你的项目路径
```

2. 创建虚拟环境（可选但推荐）
```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
```

3. 安装依赖包
```bash
pip install -r requirements.txt
```

### 2. 使用 Conda 或 UV 管理虚拟环境（可选）

**使用 Conda：**
```bash
# 创建虚拟环境
conda create -n volunteer-scheduler python=3.10
# 激活虚拟环境
conda activate volunteer-scheduler
# 安装依赖
pip install -r requirements.txt
```

**使用 UV（更快的替代方案）：**
```bash
# 创建虚拟环境
uv venv venv
# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
# 使用 UV 安装依赖（比 pip 快 5-10 倍）
uv pip install -r requirements.txt
```

### 3. GUI 界面额外配置（可选）

如果你想使用图形界面版本，需要额外安装 PyQt5：
```bash
pip install PyQt5>=5.15.0
```

> **Windows 用户：** 系统自带中文字体，无需额外处理
> 
> **Mac/Linux 用户：** 需要先安装系统中文字体包

### 4. 输入文件准备

在项目的 `input/` 文件夹下放置以下 Excel 文件：

**必需文件：**
- `普通志愿者招募表.xlsx` - 普通志愿者报名表
- `内部志愿者表.xlsx` - 内部员工志愿者表
- `家属志愿者表.xlsx` - 员工家属志愿者表
- `情侣志愿者表.xlsx` - 情侣志愿者登记表
- `岗位表.xlsx` - 活动岗位及人数表
- `直接委派名单.xlsx` - 直接分配名单

**可选文件夹：**
- `input/面试打分表/` - 放置各个面试官的打分表
- `input/团体/` - 放置团体志愿者表（可多个文件）

## 第二部分：四种使用方法

### 方法一：🖥️ 命令行模式（最灵活）

最大的优势是能单独执行任意步骤，容错性最高。出问题时可重新执行某一步。

**快速执行整个流程：**
```bash
python main.py --all
```

**分步执行（推荐新手）：**
```bash
# 第1步：处理面试结果
python main.py --interview

# 第2步：进行排表
python main.py --scheduling
```

---

#### 🎯 最灵活的方式：逐个程序单独执行

如果想对每一步有最细致的控制，可以按照下面的顺序逐个执行各个程序。

**【面试结果处理阶段】**

*第1步：汇总面试打分表*
```bash
python src/interview/summarizer.py
```
**作用：** 从多个面试官的打分表中汇总数据，生成统一的面试打分表

*第2步：分离已面试和未面试人员*
```bash
python src/interview/separator.py
```
**作用：** 将已参加面试的志愿者和未参加面试的志愿者分开，为后续排表做准备

---

**【排表主程序阶段】**

*第3步：基本信息核查和元数据收集*
```bash
python src/scheduling/pre_checker.py
```
**作用：** 检查所有输入文件中的数据完整性和重复问题，收集排表所需的基本信息

*第4步：拆分正式与储备志愿者*
```bash
python src/scheduling/splitter.py
```
**作用：** 根据面试成绩和资格条件，将普通志愿者分为正式和储备两档

*第5步：家属志愿者资格审查*
```bash
python src/scheduling/family_checker.py
```
**作用：** 验证家属志愿者是否有相关内部员工关系，审查其资格

*第6步：情侣志愿者资格核查*
```bash
python src/scheduling/couple_checker.py
```
**作用：** 验证情侣志愿者之间的关系是否真实，是否可以分配到同一小组

*第7步：小组划分和组长分配*
```bash
python src/scheduling/group_allocator.py
```
**作用：** 根据岗位需求和特殊角色要求，建立小组结构并分配组长

*第8步：生成绑定集合*
```bash
python src/scheduling/binder.py
```
**作用：** 将情侣、家属等需要绑定的志愿者聚合成一个整体，便于集中分配

*第9步：执行排表主程序*
```bash
python src/scheduling/main_scheduler.py
```
**作用：** 核心调度引擎，将所有志愿者分配到各个小组和岗位

*第10步：总表拆分和最终生成*
```bash
python src/scheduling/finalizer.py
```
**作用：** 将总表按小组拆分，生成各小组的独立表格和综合大总表

---

**额外操作（可选）：提取绑定人员详情**
```bash
python src/scheduling/binding_extractor.py
```
**作用：** 从最终排表中提取所有绑定关系人员的详细信息单独成表

**💡 何时选择这种方法：**
- 第一次尝试，想知道各步骤的具体进展
- 某个步骤出错想重新执行
- 想更细致地控制整个过程
- 想深入理解系统的运作流程

---

### 方法二：🎨 图形界面（最友好）⭐ **推荐新手**

最简单直观，不需要打命令行，点按钮即可。

**启动步骤：**

1. 打开项目文件夹
2. 双击 `gui_main.py` 或在命令行运行：
```bash
python gui_main.py
```

3. 界面打开后，按顺序点击按钮执行各步骤：
   - 汇总面试打分表
   - 分离已面试人员
   - 基本信息核查
   - 拆分正式/储备志愿者
   - 家属资格审查
   - 情侣资格核查
   - 小组划分与分配
   - 生成绑定集合
   - 执行排表主程序
   - 生成最终输出表格

4. 实时查看每步的执行进度和日志

**💡 何时选择这种方法：**
- 对命令行不熟悉
- 想直观看到进度
- 想逐步执行，观察中间结果

---

### 方法三：⚙️ 交互式菜单（简单易用）

通过菜单选择执行各个模块，类似方法一但操作更简单。

**启动步骤：**

1. 在命令行运行：
```bash
python bot.py
```

2. 按照菜单提示输入数字选择要执行的步骤：
   - 输入 `1` ~ `10` 执行对应功能
   - 输入 `h` 看帮助
   - 输入 `q` 退出

3. 按照建议顺序执行：
   - 先执行面试模块（1-2号）
   - 再执行排表模块（3-10号）

**⚠️ 注意：**
- 这种方式可能在某些环境下不太稳定
- 如果出现异常，建议切换到方法二

**💡 何时选择这种方法：**
- 介于方法一和方法二之间的选择
- 想要有序地执行但不想用图形界面

---

### 方法四：⚡ Make 快速命令（高效批处理）

当你已经熟悉系统，可以用 Make 命令快速大批量处理。但缺乏单个步骤的独立控制。

**主要命令：**

```bash
# 一键执行完整流程
make run-all

# 分步执行
make run-interview        # 面试模块
make run-scheduling       # 排表模块
make extract-bindings     # 提取绑定人员

# 个别工具使用
make merge ARGS="file1.xlsx file2.xlsx output.xlsx"      # 合并 Excel
make sort ARGS="file.xlsx 列名"                         # 排序
make dedup ARGS="file.xlsx 列名1,列名2"                 # 去重
make split ARGS="file.xlsx 列名"                        # 拆分文件
make compare ARGS="main.xlsx compare.xlsx key_col"      # 对比文件
make join ARGS="file1.xlsx file2.xlsx key_col col1,col2" # 合并字段
```

**💡 何时选择这种方法：**
- 已熟悉系统，想快速执行
- 想一键完成整个任务
- 想使用单个工具进行数据处理

## 第三部分：输出结果

| 文件夹 | 说明 |
|------|------|

## 常见问题

**Q: 能不能只执行某一个步骤？**
A: 可以，用方法一最灵活。比如只想处理面试结果就执行 `python main.py --interview`

**Q: 执行出错了怎么办？**
A: 
1. 检查输入文件是否放在 `input/` 文件夹
2. 查看 `logs/` 文件夹中的日志文件了解错误原因
3. 用方法一的分步执行定位具体是哪一步出问题
4. 修正输入数据后重新执行该步骤

**Q: 推荐用哪种方法？**
A: 
- **首次使用：** 方法二（GUI）- 最直观
- **熟悉系统后：** 方法一（命令行）- 最灵活
- **批量处理：** 方法四（Make）- 最高效

**Q: 需要每次都把四种方法都执行一遍吗？**
A: 不需要，四种方法是**并列的替代方案**，选一种即可完成整个处理流程。
